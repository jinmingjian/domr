ObjectManager Enterprise FAQ

 

 

What does ObjectManager Enterprise do? Why should I use it?

 

ObjectManager Enterprise is advanced version of the freely available Object Manager. It has features that allow programmers to do all db4o related operations from within Eclipse IDE. So a programmer can browse a db4o database, query it, access support cases, request for pairing, check latest db4o downloads and do many more things without having to open different applications. This enhances programmer productivity and gives them a nice, new experience of programming with db4o.

 

 

db4o is available in two flavors, what about ObjectManager Enterprise?

 

The ObjectManager Enterprise is also available in two flavors.  For .NET programmers it is available as a Visual Studio .NET 2005/2008 plug-in. For Java programmers the ObjectManager is an Eclipse plug-in.

 

 

Can I use ObjectManager Enterprise while I debug another application in Eclipse?

 

Yes, you can. The ObjectManager Enterprise for Java has its own perspective which can be conveniently switched with other perspectives. So you can comfortably switch between the ObjectManager perspective and any other perspective.

 

Connecting and Browsing

 

 

How do I connect to a db4o database through ObjectManager Enterprise?

 

ObjectManager Enterprise can connect to db4o in two modes, local and remote. In local mode, browse and select the db4o database file and click open. In remote mode, enter the host, port, username and password and click connect. In either case, you can even choose from a list of recent connections by clicking it.

 

 

How do I see all classes (along with their attributes) saved in my db4o database?

 

On connecting to a database, a db4o Browser opens and lists all classes saved in the database. User can choose the Flat View where all classes are listed or the Package View where all packages are listed.

 

User can select a class by clicking and expanding the hierarchy by clicking on '+' sign. User can navigate to any attribute at any level in the hierarchy.

 

 

How do I see all objects of a particular class in my database?

 

User can select a class as described above and right click on the selected class. Click on the ‘View All Objects’ menu that pops up. A new tab displaying all objects of the selected class in a table view opens. To examine, expand and navigate click on an object in the table view. A tab showing details of the selected object in a tree grid view appears below the table.

 

 

Database, Class and Object Properties

 

 

How do I see database size (and other db properties)?

 

A Property Viewer view is shown at the bottom of the plug-in. Click on the Database Properties tab. All database properties such as database size, free space, number of different classes whose objects are saved in the database can be seen here.

 

Note: These properties are not seen for a remote connection.

 

 

How can I find the names of indexed attributes of a class?

 

In the Property Viewer view there is a separate tab which shows the Class properties. Select a class in db4o Browser and click on Class Properties tab in this view. A table shows all attributes of a class along with access specifier (public, non-public) and index details. This tab also shows the count of objects of the selected class in the database.

 

 

How can I find uuid and version no. of an object in the database?

 

The Property Viewer view also contains a Object Details tab which shows all the details about an object. Select an object and click on the Object Properties tab in this pane. A table shows uuid and version no. for the selected object.

 

 

Ad-hoc Querying

 

 

How to form a query?

 

To build a query, drag and drop attributes of a class from the tree view to the Query Builder pane. Set constraints on attributes by providing a condition and value. Multiple constraints can be joined with logical operators.

 

For complex queries, user can form Parenthesized Query Expressions and join them by logical operators.

 

Click on Run Query to see the results.

 

 

Which elements can participate in a query?

 

Only class attributes of primitive datatype (String, numeric, Boolean, date etc) can participate in a query. These attributes can belong to any level in the reference hierarchy.

 

However, attributes of complex types, collections and primitive elements inside collections cannot participate in the query.

 

 

What are View Attributes?

 

View Attributes is the list of attributes displayed in the table view of query result. Users can drag and drop attributes from the tree view of a class to View Attributes. On clicking Run Query, these attributes are seen in the result table.

 

 

How to form a nested or parenthesized query?

 

For nested queries, users can form Parenthesized Query Expressions and join them with logical operators. Parenthesized Query Expressions are formed by dragging and dropping attributes from class view to the Query Builder and then associating conditions and providing values.

 

Updating, Deleting objects

 

 

How to edit and update an object?

 

Editing is possible in both, tree view and table view. Select object to be edited. Edit any attribute value by double clicking on the cell. The new value appears and is even reflected in the other view. Click the Save button to update the object in the database.

 

Note: Edit and save one object at a time. Do not edit another object before saving the first.

 

 

How to delete an object?

 

Select the object to be deleted in the table view. Click on Delete button.

 

 

 

Browser functions

 

How can I find the latest db4o versions available for download?

 

Click Tools > ObjectManager Enterprise > db4objects Downloads. This displays a page showing latest available versions of db4o products.

 

 

How can I access professional db4o support?

 

db4o provides professional support to licensed users. Through this, users can raise Support Cases and expect a guaranteed response by email within 24 hours. To access this click OME > Support Cases or directly click   button on the toolbar.

 

 

What is XTreme Connect? How can I access it?

 

XTreme Connect is the next level of professional support. It allows users to pair up and share his application with a support engineer. The support engineer can look at the user's code and his problem first hand and discuss it over Internet Telephony tools. It’s almost like support through video conferencing.

 

To access XTreme Connect click OME > XTremeConnect or directly click   button on the toolbar.

 

Can I start multiple instances of ObjectManager Enterprise on my machine?

 

Yes. It’s possible to start multiple instances of ObjectManager Enterprise on one machine. However, do not connect to the same database in local mode from more one instance.
